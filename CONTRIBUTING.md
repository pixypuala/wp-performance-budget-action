# Contributing

Thank you for improving this project.

## Before opening code

- Search existing issues and discussions.
- For large changes, propose the problem and contract first.
- Do not publicly disclose security vulnerabilities; follow `SECURITY.md`.

## Local setup

Provide exact prerequisites and commands here. The documented path must work from a clean clone.

## Pull requests

Include problem, approach, alternatives, risks, tests, screenshots for UI, accessibility impact, migration impact, and rollback notes.

## Quality

Run the same checks as CI. Add regression tests for bug fixes. Keep changes scoped and document user-visible behavior.

## Conduct

Follow `CODE_OF_CONDUCT.md`.
